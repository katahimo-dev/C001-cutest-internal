### 各モジュールの詳細

#### 認証基盤モジュール（Auth Module）

現行GASの認証部分を強化し、全モジュールで共通利用できる認証基盤を構築します。

| 機能 | 現行GAS | 新システム |
|------|---------|-----------| 
| ログイン方式 | メールアドレス + パスワード | 同左（Firebase Authentication） |
| パスワード保存 | Salt付きSHA-256ハッシュ | Firebase標準（bcrypt相当） |
| セッション管理 | 7日間ローリング有効期限 | 同左（Firebase Auth Token） |
| パスワードリセット | メール認証コード | 同左（Firebase標準機能） |
| 二段階認証 | 未対応 | **対応可能**（オプション） |

#### データ基盤モジュール（Data Module）

スタッフ・顧客・スケジュール情報を一元管理する共通データ基盤です。

| 機能 | 概要 |
|------|------|
| **スタッフマスタ** | 氏名、住所、連絡先、権限等（手動登録 or CSVインポート） |
| **顧客マスタ** | 顧客情報の一元管理（手動登録 or 外部連携アダプター経由） |
| **スケジュール管理** | 訪問予定の管理（手動登録 or 外部連携アダプター経由） |
| **操作ログ** | 誰が・いつ・何をしたか記録（監査証跡） |

```
【データ取り込みパターン】

パターンA（キューテスト社）:
  RESERVA CSV → 顧客マスタ
  Googleカレンダー → スケジュール管理

パターンB（他社例）:
  手動登録 → 顧客マスタ
  手動登録 → スケジュール管理

パターンC（将来）:
  予約機能モジュール → 顧客マスタ + スケジュール管理
```

```

└─────────────────────────────────────────────────┘
```

**メリット：**

### フェーズ2：SaaS展開フェーズ（将来構想）

#### SaaS料金プラン（将来構想）

#### レベニューシェアモデル

キューテスト社経由で新規顧客を獲得した場合、サブスクリプション収益を両社で分配します。

**収益分配の方針（案）：**

| 収益対象 | かたひも | キューテスト社 | 備考 |
|----------|---------|--------------|------|
| キューテスト社経由の新規顧客収益 | 50% | 50% | 経費（サーバー費・保守費）控除後の利益を折半 |
| キューテスト社自社の運用費相殺 | — | 100% | レベニューシェア分で保守費用を相殺 |

**※ 利益 ＝ 売上 − （サーバー費用 ＋ 保守メンテナンス費用）**

**レベニューシェアの開始条件と詳細：**
- キューテスト社が本社িস্টেম を正常に本番稼働中であること
- キューテスト社経由で紹介された顧客が契約を締結した場合に発生
- 収益分配は毎月末に集計し、翌月中に双方が確認・合意した上で実施
- 具体的な分配率は、SaaS展開フェーズへの移行時に別途合意する

**キューテスト社へのメリット：**

| 項目 | 内容 |
|------|------|
| **運用管理費の相殺** | レベニューシェア分で保守費用を相殺 |
| **紹介収益** | 業界内の横展開で継続的な収益 |
| **システム進化** | 他社導入時の機能強化が自社にも還元 |
| **業界貢献** | 保育業界全体のDX推進に寄与 |

### なぜこのモデルなのか

```
従来のSI開発:
  発注者 → 完成品を受け取るだけ → 開発者との関係は終了

共創モデル:
  パートナー → 一緒に育てる → 長期的なWin-Winの関係
```

**キューテスト社は単なる「お客様」ではなく、「共同開発パートナー」です。**

- 開発初期の投資を担っていただく代わりに
- 将来の横展開における収益をシェア
- システムの進化を共に享受

これにより、**「作って終わり」ではなく「共に成長する」関係**を構築します。

# よくあるご質問 (FAQ)

## 契約・プランについて

### Q1. 途中でプラン変更は可能ですか？
はい、月単位で変更可能です。繁忙期はプランA（標準）で開発を加速し、落ち着いたらプランC（スモール）に切り替えるなど、貴社の状況に合わせて柔軟に対応いたします。

### Q2. 開発が長引いた場合、追加料金は発生しますか？
準委任契約のため、稼働時間に応じたお支払いとなります。ただし、当方の責による遅延（バグの多発や設計ミスによる手戻り等）の場合は、追加料金なく誠意を持って対応いたします。

### Q3. 解約したくなった場合はどうすればよいですか？
1ヶ月前までの書面通知により、いつでも解約が可能です。成果物はその時点での最新版を全て納品いたします。

## 開発・技術について

### Q4. 他の開発者や会社に引き継ぎできますか？
はい、可能です。ソースコード、設計書、データベース定義書など、全ての成果物は貴社に帰属します。GitHubやGoogle Driveを通じて常に最新の状態を共有しますので、いつでも他社や内製チームへスムーズに引き継ぎが可能です。

### Q5. 他案件との並行はありますか？
品質を確保するため、同時並行は最大2案件までに限定しております。貴社プロジェクトを優先的に対応いたしますのでご安心ください。

### Q6. セキュリティ対策は大丈夫ですか？
はい、銀行や大手企業も採用するGoogleのセキュリティ基盤（Firebase Authentication等）を使用します。通信は全て暗号化（SSL）され、パスワードもハッシュ化して保存されます。また、開発者の端末には二段階認証やディスク暗号化を適用し、万全の体制で臨みます。

---

## 運用・サポートについて

### Q7. 障害が発生した場合はどうなりますか？
保守プランにご加入いただいている場合、Slackやメールでご連絡いただければ、プランの規定時間内で迅速に対応いたします。サーバーレス構成（Firebase）のため、サーバーダウンのリスク自体が非常に低いのが特徴です。

### Q8. データはバックアップされていますか？
はい、Cloud Firestoreの自動バックアップ機能により、データは保護されています。また、重要なデータ（出勤簿等）は定期的にCSVとしてGoogle Driveへエクスポートする機能も実装可能です。

### Q9. スタッフが増えた場合、追加費用はかかりますか？
本システムの利用料としての追加費用はかかりません（何名でも利用可能です）。ただし、FirebaseやGemini APIの実費（従量課金）は利用者数に応じて微増する可能性がありますが、100名規模までは非常に低廉（数千円程度）に収まる設計です。
