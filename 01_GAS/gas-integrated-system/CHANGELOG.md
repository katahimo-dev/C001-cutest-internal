# 更新履歴 (Release Notes)

## [Ver. 1.0.1] - 2026-04-16

### ご要望
- 2026年度に切り替わったタイミングで、年度連携が止まる事象を解消したい。
- 毎年度の手動設定変更を減らし、運用負荷を下げたい。
- 修正期限の運用を見直し、7日前までではなく当月分を編集できるようにしたい。

### 追加機能
- 会計年度の自動判定ヘルパーを追加。
  - getCurrentFiscalYearString_
- 年度固定が必要な場合に備えた上書き設定を追加。
  - YEAR_OVERRIDE
- Google カレンダーからの直接同期機能を追加。
  - `syncCalendarToTimesheet()`: カレンダー予定を取得して出勤簿へ直接反映
- ルート情報フィールド（移動時間・移動距離・出勤/退勤距離）を Web フォームに追加。
  - 列 H, Q, AG, AH, AI, AJ に対応
- 事務作業の内容説明フィールド（事務作業1・2）を選択肢からテキスト入力に変更。
  - カレンダーから自動取得した任意の文字列に対応

### 改善・変更
- `YEAR: '2025'` の固定設定を廃止し、4月始まりの会計年度を自動算出するよう変更。
- 必要時のみ `YEAR_OVERRIDE` で固定年度に切り替えられる運用に変更。
- 修正期限の判定を「7日前まで」から「当月（1日〜末日）まで」に変更。
- 期限超過時のメッセージを当月基準の文言に更新。
- 管理者ユーザーは対象スタッフを選択し、全員分のスケジュールを確認・修正できるように変更。
- カレンダー連携の流れを勤怠集計シート経由から出勤簿ファイルへの直接反映に変更。
  - Web フォームで確認・修正後、出勤簿本体に即座に保存


### 技術的な修正
- `config.js` の年度設定初期化を見直し、初期化順序に依存しない実装へ変更。
- 年度算出ロジックを共通化し、翌年度以降もコード変更なしで動作する構成に修正。
- `doget.js` の保存時バリデーションを当月開始日判定へ変更し、前月以前を編集不可に統一。
- `index.html` の事前チェックも同一ロジックへ合わせ、画面とサーバーの判定不一致を解消。
- `doget.js` に管理者判定・対象スタッフ解決ロジックを追加し、検索/保存時の権限制御を強化。
- `index.html` に管理者専用のスタッフ選択UIを追加し、対象スタッフ指定をサーバーへ連携。
- `doget.js` に `syncCalendarToTimesheet()` を追加し、カレンダー連携から出勤簿への直経路を統一。
  - に従属関数 `findTargetRow_()`、`readRowData_()`、`writeRowData_()` を追加し、行操作を抽象化。
- INPUT_COLUMNS を拡張し、ルート情報列（H, Q, AG, AH, AI, AJ）の読み書きに対応。
  - 従来の 「只読」属性（C, L, U）を削除し、ユーザー編集を許可。
- `index.html` から冗長な勤怠集計データ表示・保存関数を削除。
  - `loadAttendanceSheetData()`, `displayAttendanceData()`, `saveAttendanceSheetData()`、および関連 UI セクションを廃止。
- `index.html` の事務作業フィールド（col_X, col_AA）を select から text 入力に変更し、カレンダーからの任意文字列対応。
- ライブラリ（gas-root-serach）を作成し、カレンダー取得→ルート計算→出勤簿行データ生成ロジック一元化。
