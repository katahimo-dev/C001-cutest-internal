# gas-integrated-system 詳細仕様書

## 1. 概要
本プロジェクトは、スタッフが自分のスマートフォンから個人の出勤簿（Googleスプレッドシート）を直接修正・入力するためのWebアプリケーション（Google Apps Script Web App）です。

## 2. 主な機能
### 2.1. 勤怠修正UI (`index.html`, `doget.js`)
- **日付選択**: 指定した日付の勤務データを個人の出勤簿からリアルタイムに取得して表示します。
- **入力項目**: 始業/終業時刻、天候、作業記録（作業内容・時間）、買物代行、備考などを入力可能です。
- **セレクトボックス連携**: スプレッドシート側の「データの入力規則」と同期し、選択肢（天候や作業内容など）を動的にプルダウンに反映します。
- **修正箇所の可視化**: Webアプリから変更されたセルは、スプレッドシート上で背景色が薄赤色 (`#fce4e4`) に変更され、視覚的に追跡可能になります。

### 2.2. 権限・アクセス管理
- **スタッフ認証**: 実行者のGoogleアカウント（Email）を「Staff」名簿と照合し、一般スタッフは本人専用の出勤簿ファイルのみを操作できるように制御します。
- **管理者モード**: 管理者メールとして登録されたユーザーは、全スタッフから対象者を選択して閲覧・修正できます。
- **修正期限の制限**: 当月（1日〜末日）の記録のみ修正を許可し、前月以前のデータは期限切れとしてエラーを表示します。
- **読み取り専用設定**: 訪問先名、など一部の重要項目はWebアプリからの変更を許可しない設定が可能です。

### 2.3. 運用補助機能
- **アプリURL送信**: 管理者が「Staff」シートでチェックを入れたスタッフ宛に、本アプリのアクセスURLをメールで一括送信する機能を備えています。
- **スタッフ同期 (`STAFF_SYNC.js`)**: 外部のマスターデータとスタッフ情報（名前、Email、退職状況）を同期します。

## 3. 技術仕様
- **フロントエンド**: Vanilla JS と CSS (Responsive Design) により、モバイル端末に最適化されたUI。
- **バックエンド**: GAS (`doGet`, `google.script.run`) によるサーバーサイド処理。
- **データストア**: 各スタッフのGoogleスプレッドシート（`スタッフ名_出勤簿_XXXX年度`）をDBとして利用。

## 4. セキュリティ
- Googleの「自分として実行」または「アクセスしているユーザーとして実行」の権限設定に基づき、スタッフ間のデータ混線を防止。
- メール送信やファイル検索において、退職済みのスタッフを除外するロジックを搭載。

## 5. 設定項目
- `YEAR`: 対象年度の定義
- `INPUT_COLUMNS`: スプレッドシートのカラムとUIラベルのマッピング定義（ReadOnly設定含む）
