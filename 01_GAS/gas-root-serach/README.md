# gas-root-serach

Googleカレンダー予定を解析し、スタッフ別の移動ルート時間/距離を算出して集計・通知するGASプロジェクト。

## 目的

- 訪問予定から移動コストを自動算出。
- 集計シートの定常更新。
- 日次通知と勤怠転記連携の基盤提供。

## 主な機能

- `main.js`
  - カレンダー予定の取得とスタッフ別振り分け。
  - 種別判定（予約確定/新規/イベント/事務）。
  - 住所・座標処理、ルート算出、集計書き込み。
- `lineworks.js`
  - LineWorks Bot通知。
- ルート計算
  - 出勤（自宅→1件目）
  - 訪問間移動（N件目→N+1件目）
  - 退勤（最終訪問→自宅）
- 例外処理
  - 招待者なしイベントのオーナー割当。
  - 事務予定を挟むケースの距離計算補正。
  - 期間限定住所の優先適用。

## 入出力

- 入力
  - Google Calendar
  - 顧客CSV（`Kokyaku_*.csv`）
  - スタッフマスター
- 出力
  - ルート集計スプレッドシート（月別シート）
  - 勤怠集計スプレッドシート（運用追加分）
  - LineWorks通知

## 運用

- 日次トリガーで自動実行。
- 特定日の再実行は手動関数で実施。
- 月次大量処理はタイムアウト回避のため分割実行を推奨。

## 必須設定

- `CONFIG` の各ID（顧客フォルダ、スタッフSSなど）。
- Script Properties のLineWorks認証情報。
