# gas-childcare-daily-report

スタッフ別出勤簿の一括生成と、ルート集計データからの勤怠自動転記を行うGASプロジェクト。

## 目的

- 出勤簿ファイル作成作業の自動化。
- 日次の勤怠転記作業の省力化。
- スタッフマスタ同期による運用ミス低減。

## 主な機能

- `attendance.js`
  - 出勤簿テンプレート複製。
  - 年度フォルダへのスタッフ別ファイル生成。
  - 保護設定のコピー。
  - 退職者除外。
  - 長時間実行時の再開制御。
- `transfer.js`
  - ルート集計ファイルから日付指定でデータ取得。
  - スタッフ名・日付キーで出勤簿へ転記。
  - 訪問複数件、通勤/退勤距離に対応。
  - 日次自動実行・手動実行。
- `staffinfomation.js`
  - スタッフ情報の同期更新。

## 必須データ

- スタッフ一覧シート。
- `出勤簿テンプレート` スプレッドシート。
- ルート集計スプレッドシート。
- 年度出力先フォルダ（例: `2025出勤簿`）。

## UIメニュー

- スタッフ情報更新
- 出勤簿入力（日付指定 / 明日分自動）
- 出勤簿作成（年度）

## 運用手順

1. スタッフ情報を同期。
2. 年度切替時に出勤簿を一括作成。
3. 日次で勤怠転記を実行（トリガー推奨）。
4. 必要時に日付指定で再転記。

## 注意点

- ルート集計の列構成変更時は `CONFIG_TRANS` の調整が必要。
- 共有ドライブでは保護設定コピーの権限エラーに注意。
- 詳細仕様は `specification.md` を参照。
