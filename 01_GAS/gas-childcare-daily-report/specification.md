# gas-childcare-daily-report 詳細仕様書

## 1. 概要
本プロジェクトは、スタッフ別の出勤簿（Googleスプレッドシート）を一括生成し、日々のルート集計データから勤怠情報を自動転記するためのGoogle Apps Script（GAS）システムです。

## 2. 主な機能
### 2.1. 出勤簿一括作成 (`attendance.js`)
- **テンプレートからの複製**: 指定された「出勤簿テンプレート」をコピーし、年度（4月〜3月）ごとの月別シートを持つスタッフ別ファイルを作成します。
- **重複スキップ**: すでに同名のファイルが存在する場合は作成をスキップし、既存ファイルへのリンクのみ更新します。
- **保護設定の同期**: テンプレートに設定されているシート保護（編集不可範囲）を、作成された各スタッフのシートに正確にコピーします。
- **退職者判定**: スタッフ名簿の退職日を参照し、対象年度より前に退職している場合は作成対象外とします。
- **タイムアウト対策**: 処理時間が5分を超える場合に一旦停止し、次回の実行で続きから開始できる仕組みを備えています。

### 2.2. 勤怠データ自動転記 (`transfer.js`)
- **ルート集計との連携**: 外部ファイル「ルート集計」から特定の日付のデータを取得します。
- **自動マッピング**: スタッフ名と日付をキーにして、各スタッフ個別の出勤簿ファイルの該当箇所（C列:訪問先、D列:開始、E列:終了など）に値を書き込みます。
- **多様な転記パターン**: 1日の複数回の訪問（1件目〜3件目）や、通勤/退勤距離の転記に対応しています。
- **手動・自動実行**: 任意の日付を指定して実行する機能と、トリガーによって翌日分を自動転記する機能を備えています。

### 2.3. スタッフ情報同期
- `staffinfomation.js` を通じて、外部のマスターデータからスタッフ情報を取得・更新します。

## 3. システム構成
- **マスターシート**: スタッフ一覧と、各年度の出勤簿ファイルへのハイパーリンクを保持します。
- **テンプレート**: `出勤簿テンプレート` という名称のスプレッドシートを使用します。
- **出力先**: 会計年度ごとのフォルダ（例: `2025出勤簿`）に各スタッフのファイルが保存されます。

## 4. UI・メニュー
スプレッドシートの上部メニューに以下の項目を追加します：
- **🟩スタッフ情報更新**: 最新情報を取得する
- **🟦 出勤簿入力**: 出勤簿ファイルへ転記（日付指定）、明日の分を自動転記
- **🟧 出勤簿作成（年度）**: 出勤簿一括作成

## 5. 注意事項
- ルート集計ファイルのフォーマット依存性が高いため、カラム位置が変更された場合は `CONFIG_TRANS` の調整が必要です。
- 共有ドライブ環境での実行時は、ファイルの保護設定コピー時に権限エラーが発生しないようエラーハンドリングが行われています。
