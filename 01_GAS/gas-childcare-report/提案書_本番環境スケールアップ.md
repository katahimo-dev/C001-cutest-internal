# 【提案書】ベビーシッター日報システム 本番環境スケールアップのご提案

## 1. 提案の要旨（エグゼクティブサマリー）
現在のシステムは、仮説検証を最速で行うための「試作品（プロトタイプ）」として構築しました。
事業が拡大し、ユーザー数がこれから増えていくこのタイミングで、システムの中身を**「試作品」から「プロ仕様の製品版」へ作り変える（リニューアルする）**ことをご提案します。

これにより、ユーザーが増えても**「止まらない」「遅くならない」**安定したサービスを提供し、現場のベビーシッター士さんが**「使いやすい！」**と感動する体験を実現します。

## 2. なぜ今、投資が必要なのか？（現状のリスク）
現在のシステムは、Googleスプレッドシートに付属している簡易的なプログラム機能（Google Apps Script）で動いています。これは低コストで手軽に作れる反面、本格的な業務利用には以下のリスクがあります。

*   **🚨 「お店に行列ができて入れない」状態になるリスク**
    現在の仕組みは、一度にたくさんの人が使うことを想定していません。ユーザーが増えると「保存ボタンを押しても反応しない」「エラーが出る」といった事態が起きやすくなり、現場の信頼を失ってしまいます。
*   **📉 「Excelが重くて動かない」状態になるリスク**
    今のデータの保存先はスプレッドシート（いわばExcelファイル）です。データが数千件、数万件と増えていくと、動作が極端に遅くなり、毎日の日報作成に時間がかかるようになります。
*   **🚫 拡張性の限界**
    今後、「さらに高度なAI分析をしたい」「スマホアプリ版を利用したい」と思った時に、現在の土台のままでは対応できず、結局ゼロから作り直すことになります。

*   **🔓 セキュリティと予期せぬデータ事故のリスク**
    *   **パスワードの平文管理**: ログインIDとパスワードが「文字そのまま」で保存されています。現代のシステムではあり得ない状態で、万が一、管理者の個人アカウントが不正アクセスを受ければ、全スタッフのパスワードが即座に流出し、なりすまし等の被害に直結します。
    *   **個人アカウントへの依存（事業継続リスク）**: システム全体が経営者個人のGoogleアカウントに紐付いています。もしアカウント凍結や事故で管理者がログインできなくなると、**会社全体の業務が即ストップ**し、誰も復旧できなくなる「単一障害点」を抱えています。
    *   **操作ログが残らない**: 「誰が・いつ・誰のデータを・見た/編集したか」という記録（監査ログ）が確実には残りません。
    *   **データの脆さ**: データベースが「表計算ソフト」であるため、操作ミスで列ごとデータを消してしまったり、上書きしてしまう事故が起きやすい構造です。

## 3. 解決策：IT大手も採用する「プロ仕様」への移行
ユニコーン企業や急成長スタートアップが標準的に採用している、最新の仕組み（モダン・アーキテクチャ）に刷新します。

| 改善ポイント | 今の状態 (Before) | 新しい状態 (After) | イメージ例え |
|---|---|---|---|
| **画面の使い心地**<br>(UI/UX) | Webページを毎回読み込むため、カクカクして待ち時間が長い。 | **スマホアプリのようにサクサク動く**。画面切り替えが一瞬。 | ガラケー ➡ **スマートフォン** |
| **データの保存場所**<br>(データベース) | スプレッドシート。<br>手軽だが、量が増えると遅くなる。 | **専用のデータベース**。<br>何百万件データがあっても一瞬で検索可能。 | 手書きの大学ノート ➡ **自動検索システム付きの巨大倉庫** |
| **システムの基盤**<br>(インフラ) | 無料版の簡易スペース。<br>処理能力に厳しい制限がある。 | **クラウドサーバー**。<br>アクセス集中時、自動で能力を拡張して対応する。 | 軽自動車 ➡ **大型トラック** |
| **安全性・管理**<br>(セキュリティ) | URLを知っていれば誰でも見れる。<br>操作ログが残らない。 | **厳密な権限管理**。<br>「誰がいつ何をした」か全て記録される。 | 鍵のない部屋 ➡ **生体認証・入退出記録付きの金庫室** |
| **認証・事業継続性**<br>(ID管理) | パスワードが丸見え（平文）。<br>社長個人の無料枠に依存し、何かあれば即停止。 | **弊社運用のクラウド基盤**。<br>御社での管理は不要。専門チームが安定稼働を保証。 | 家族経営の個人商店 ➡ **大手ショッピングモールのテナント** |

### 【重要】特定されたリスクへの具体的対処
先述の「パスワード平文管理」や「個人アカウント依存」といった重大リスクは、以下のように技術的に解決されます。

*   **パスワード問題の解決**:
    Googleが提供する認証基盤（Firebase Authentication）を採用します。これにより、パスワードは**高度に暗号化**されて保存されるため、開発者ですら見ることはできません。また、パスワードを使わず「Googleアカウントでログイン」する方式も選択可能になり、セキュリティレベルが劇的に向上します。
*   **個人アカウント依存（単一障害点）の解消**:
    現在の「経営者個人のドライブ」から、弊社が管理・運用する**商用クラウド基盤（Google Cloud等）**へとシステム全体を移行します。
    インフラ管理は全て弊社が行うため、御社でのサーバー管理やアカウント維持の手間は一切不要になります。
    **システム障害**や**外部連携（LINEなど）の突然の不具合**といった不測の事態においても、弊社のエンジニアが原因特定から復旧までを責任を持って担当します。これにより、事業が個人アカウントのトラブル等で「突然止まる・直せない」というリスクを完全に排除できます。


## 4. ビジネスへの効果

1.  **現場の生産性が向上**: 操作の待ち時間がほぼゼロになり、ベビーシッター士さんが本来の業務（子供と向き合う時間）に集中できます。
2.  **AI精度の向上**: データをきちんとした形式で整理・保存することで、AIがより正確に学習・分析できるようになります。
3.  **本業への完全集中**: システム保守やエンジニア採用といった「本来の事業ではない業務」に時間を割く必要がなくなり、経営資源を事業拡大や人材育成に100%集中できます。

## 6. (補足) 専門用語ミニ解説

経営判断の参考として、見積書などで出てくる用語を解説します。

*   **Google Apps Script (GAS)**:
    Googleが提供している、スプレッドシートなどを操作するための簡易プログラム。無料で手軽ですが、本格的なアプリ開発には「処理速度」や「制限」の面で不向きです。
*   **React / Next.js (リアクト / ネクスト・ジェイエス)**:
    Facebook(Meta)が開発した、Web画面を作るための技術。InstagramやNetflixのような「サクサク動く」画面を作れます。
*   **Google Cloud / Firebase (グーグルクラウド / ファイアベース)**:
    Googleが提供する「プロ向け」のシステム開発基盤。世界中のアクセスに耐えられるサーバーやデータベース機能を提供しており、これを使うことでイチからサーバーを買う必要がなくなります。
*   **スケーラブル (Scale-up)**:
    ユーザーが増えても、サーバーの台数を自動で増やしたりして、問題なく対応できる能力のこと。「拡張性がある」という意味です。
