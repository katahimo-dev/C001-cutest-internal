# gas-integrated-ops-v2 詳細仕様書

## 1. 概要

本プロジェクト **gas-integrated-ops-v2** は、以下の3つのGASプロジェクトを統合した、保育施設スタッフの勤務管理システムです：

- **gas-childcare-daily-report** → 出勤簿の自動生成・転記機能
- **gas-integrated-system** → スマートフォンからの勤務修正UI  
- **gas-root-serach** → Googleカレンダー連携

### 改善点

従来のシステムでは、スプレッドシートに事前計画データを記入する必要がありました。本統合版では、以下のような改善が加えられています：

1. **カレンダー由来データの自動取得**: Googleカレンダーから予定を直接読み込み
2. **月間集約シート**: 複数スタッフの1か月分のデータを1ファイル内に集約
3. **当日修正対応**: スタッフが当日～7日前まで修正可能
4. **事務作業の自動判定**: カレンダータグ（`[事務]`）で自動判定＋手修正可
5. **管理者確定後の個票生成**: 確定後に各スタッフ用の個別出勤簿を自動生成

## 2. 主な機能

### 2.1. カレンダー同期
- **入力**: Googleカレンダー（全社カレンダー）
- **処理**: 予定タイトルのタグで種別を判定
  - `[予約確定]` → 訪問予定
  - `[新規]` → 新規顧客
  - `[イベント]` → 会議・研修
  - `[事務]` → 事務作業フラグ
- **出力**: 月間集約シートの該当行に転記

### 2.2. Web UI（スマートフォン対応）
- **スタッフ機能**:
  - 日付選択 → 该当データを表示
  - 修正（時刻・天候・作業内容等）→ 月間集約シートに保存
  - 修正期限: 7日前まで
- **管理者機能**:
  - カレンダー同期トリガー
  - 月間集約シート → 個票生成・更新
  - 月末確定

### 2.3. データ構造

#### 月間集約シート
- **1ファイル**: 年度ごと
- **複数シート**: 月別（`202504` ～ `202503` など）
- **行構成**:
  - 1行目: ヘッダー
  - 2行目以降: 日付×スタッフ別

#### 月間集約シートの列構成

| 列 | 項目 | 説明 |
|----|------|------|
| A | 日付 | YYYY-MM-DD |
| B | スタッフ名 | 個票と対応 |
| C-E | 訪問先①/開始/終了 | 1件目の訪問 |
| F-H | 訪問先②/開始/終了 | 2件目の訪問 |
| I-K | 訪問先③/開始/終了 | 3件目の訪問 |
| L-N | 作業内容/開始/終了 | 事務作業等 |
| O | 天候 | 選択式 |
| P | 事務フラグ | Yes/No |
| Q | 確定者 | 管理者名 |
| R | 確定日時 | YYYY-MM-DD HH:MM:SS |

### 2.4. ワークフロー

1. **日次**: カレンダー予定を月間集約シートに自動転記（朝または管理者トリガー）
2. **当日～7日前**: スタッフがWeb UIから時刻・作業内容等を修正
3. **月末**: 管理者がデータを確定
4. **確定後**: 各スタッフ用の個別出勤簿ファイルを自動生成・更新

### 2.5. セキュリティ・権限

- **スタッフ認証**: Email ベースで個人の修正範囲を制限
- **修正期限**: 7日以内のみ修正可
- **readonly**: 訪問先名等、管理者設定項目は修正不可
- **管理者機能**: 管理者ユーザーのみアクセス可（別途実装推奨）

## 3. 技術仕様

### 3.1 デプロイメント

- **プロジェクト名**: gas-integrated-ops-v2
- **ファイル構成**:
  - `appsscript.json` - GAS マニフェスト
  - `config.js` - 共通設定（列定義、定数等）
  - `doget.js` - Web UI エントリ・データ処理
  - `index.html` - フロントエンド
  - `STAFF_SYNC.js` - スタッフマスター同期
  - `calendar_sync.js` - カレンダー読込・集約シート転記
  - `attendance_transfer.js` - 個票生成・転記（未実装）

### 3.2 外部依存

- **Google Calendar API**: カレンダー予定取得
- **Google Drive API**: ファイル検索・作成
- **Google Sheets API**: スプレッドシート読み書き
- **Google Mail API**: メール送信（URL配布用）

### 3.3 既知制限

- **ルート計算**: 本PJでは未実装（gas-root-serach 参照）
- **LineWorks通知**: オプション（` CONFIG_UNIFIED.USE_LINEWORKS` で制御）
- **タイムゾーン**: Asia/Tokyo に固定
- **会計年度**: 4月～3月に固定

## 4. 運用

### 4.1 初期セットアップ

1. `gas-integrated-ops-v2` をデプロイ
2. 月間集約シート（新規）を作成
3. `config.js` で `SUMMARY_SHEET_ID` を指定
4. Staff シートに管理者・スタッフを登録

### 4.2 日常運用

- **毎朝**: 「カレンダー同期」メニューから本日分を読み込み（自動トリガーも可）
- **当日**: スタッフがWeb UIから修正
- **月末**: 「確定」ボタン → 個票生成

### 4.3 トラブルシューティング

| 問題 | 原因 | 対処 |
|------|------|------|
| カレンダーが同期されない | カレンダーIDが誤り | config.js で CALENDAR_ID を確認 |
| Web UIで「スタッフ登録されていません」 | Email が Staff シートに未登録 | Staff シートに Email を追加 |
| 個票が生成されない | テンプレートファイルが見つからない | gas-childcare-daily-report 参照 |
